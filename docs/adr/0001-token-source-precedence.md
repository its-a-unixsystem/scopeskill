# Config and token source precedence

The `sv-cli` is an agent-first helper tool, so normal automation must not silently fall back to password login. Initial credentials may be used only during explicit `auth login` and must never be stored; normal API calls read machine-facing env-file config first, then allow `SCOPESKILL_*` environment variables to override any matching value, including `REST_REFRESH_TOKEN`. The long-lived REST refresh token and customer number live in config, while short-lived REST access tokens live in a separate disposable cache with expiry metadata, because the CLI exits after each command but should not mix transient request credentials into durable config.
