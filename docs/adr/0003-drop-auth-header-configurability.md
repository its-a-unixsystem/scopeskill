# Drop AUTH_HEADER configurability

The Scopevisio CLI hardcodes `Authorization` as the bearer header name and exposes no `AUTH_HEADER` config key or `SCOPESKILL_AUTH_HEADER` env override. The earlier code carried a configurable header name as a speculative escape hatch, but the Scopevisio Swagger (`https://appload.scopevisio.com/rest/swagger.json`) and the official "REST API first steps" article both specify the standard `Authorization: Bearer <token>`. Carrying a documented knob nobody needs costs us a permanent contract; if a real Scopevisio deployment ever requires a different header, re-adding the override is a one-line additive change.
