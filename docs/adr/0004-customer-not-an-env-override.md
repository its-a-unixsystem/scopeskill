# CUSTOMER is not a standalone env override

`SCOPESKILL_REST_REFRESH_TOKEN` overrides the configured refresh token, but `SCOPESKILL_CUSTOMER` deliberately does not exist. The customer number is sent only on the token endpoint (password-grant and refresh-grant) and must match the identity the refresh token was minted for, so varying `CUSTOMER` independently of `REST_REFRESH_TOKEN` cannot produce a working call — it just yields confusing 401s. The legitimate "switch identity for this run" use case is served by `--config` / `SCOPESKILL_CONFIG` pointing at a different scopeskill config file, where the matching `CUSTOMER` and `REST_REFRESH_TOKEN` travel together. Symmetry with other env overrides was rejected as a footgun in exchange for a smaller, harder-to-misuse public surface.
